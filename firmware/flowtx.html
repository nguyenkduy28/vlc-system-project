TX Flow - OWC Project

Tai lieu nay mo ta luong chay TX cua firmware OWC tren STM32F407, tap trung vao che do APP_BOARD_ROLE_TX_ONLY va cach cac lenh UART dieu khien TX runtime.

1. Tong quan

Luong TX trong firmware duoc chia thanh 2 phan:

  1. Khoi tao va bat phan cung phat.
  2. Chay theo nhip TIM2 de phat tung bit OOK ra TIM1.

Firmware giu nguyen cac rang buoc quan trong:

  • TIM1 tao carrier 1 MHz tren PA8.
  • TIM2 tao bit tick 100 kHz.
  • OOK:
    • bit 1 -> TIM1 CCR1 = 84
    • bit 0 -> TIM1 CCR1 = 0
  • Frame format: AA AA AA | D5 | LEN | PAYLOAD | CHECKSUM
  • Payload mac dinh: 55 A5 3C C3

2. Khoi chinh cua TX

KhoiVai tro
main.cGoi init he thong va vong lap chinh
app_tasks.cDieu phoi start/process/ISR theo board role
vlc_tx.cGi frame TX va phat tung bit OOK
app_protocol.cTao frame chuan va checksum
bsp_uart.cNhan lenh UART va in log

3. Flow khoi dong TX

main()
  -> HAL_Init()
  -> SystemClock_Config()
  -> MX_GPIO_Init()
  -> MX_DMA_Init()
  -> MX_TIM1_Init()
  -> MX_TIM2_Init()
  -> MX_ADC1_Init()
  -> MX_USART2_UART_Init()
  -> MX_DAC_Init()
  -> MX_TIM4_Init()
  -> app_tasks_init()
  -> app_tasks_start()
       -> set DAC threshold
       -> start ADC monitor
       -> start TX theo APP_BOARD_ROLE
       -> start TIM2 interrupt
  -> while(1)
       -> app_tasks_process()

4. Flow phat bit TX

Moi lan TIM2 ngat, firmware goi app_tasks_on_timer_period_elapsed(). O che do TX_ONLY, ISR chi goi vlc_tx_on_bit_tick().

TIM2 IRQ
  -> app_tasks_on_timer_period_elapsed()
      -> s_tick_count++
      -> vlc_tx_on_bit_tick()
          -> neu carrier_test_mode = 1
               CCR1 = 84
               return
          -> neu tx_enabled = 0 va single_frame_mode = 0
               CCR1 = 0
               return
          -> doc bit tiep theo MSB-first tu frame
          -> bit = 1 ? CCR1 = 84 : CCR1 = 0
          -> tang bit_index
          -> neu het frame
               reset bit_index
               tang frame counter
               neu single_frame_mode = 1
                   tx_enabled = 0
                   CCR1 = 0

5. Cau truc frame TX

+---------+---------+---------+---------+---------+----------------+-----------+
| Preamble| Preamble| Preamble|  Sync   |  LEN    |   PAYLOAD      | CHECKSUM  |
| 0xAA    | 0xAA    | 0xAA    | 0xD5    | 1 byte  | 1..N bytes     | 1 byte    |
+---------+---------+---------+---------+---------+----------------+-----------+

Checksum duoc tinh theo cong thuc:

CHECKSUM = (LEN + sum(PAYLOAD)) mod 256

Vi du payload mac dinh:

LEN = 04
PAYLOAD = 55 A5 3C C3
CHECKSUM = FD

6. Flow build frame TX

Khi khoi tao, TX lay payload chuan tu app_protocol_get_expected_payload() va build frame bang app_protocol_build_frame().

app_tasks_init()
  -> vlc_tx_init(htim1)
      -> app_protocol_get_expected_payload()
      -> app_protocol_build_frame()
      -> luu frame serialize vao buffer TX

7. Luong UART dieu khien TX

UART command duoc xu ly o foreground trong app_tasks_process(), khong xu ly trong ISR.

UART RX byte
  -> app_handle_uart_rx()
      -> gom byte thanh 1 dong lenh
      -> app_handle_command_line()
          -> neu la rtc command
               xu ly RTC
          -> neu APP_BOARD_ROLE_TX_ONLY
               xu ly TX command
          -> nguoc lai
               bao lenh khong ho tro

7.1 tx_payload

Lenh:

tx_payload 11 22 33 44 55 66 77 88

Luong xu ly:

  1. Tach tung token hex sau tx_payload.
  2. Parse tung byte bang strtoul(..., 16).
  3. Kiem tra gia tri <= 0xFF.
  4. Kiem tra khong vuot APP_MAX_PAYLOAD_LEN.
  5. Goi vlc_tx_set_payload(payload, len).
  6. Tra ve:
cmd_ok tx_payload len=8 checksum=04

7.2 tx_start

Bat che do phat lien tuc:

cmd_ok tx_start

7.3 tx_stop

Dung phat du lieu, carrier bi keo xuong 0:

cmd_ok tx_stop

7.4 tx_carrier_on

Giu carrier ON lien tuc de do kiem:

cmd_ok tx_carrier_on

7.5 tx_carrier_off

Tat carrier test mode:

cmd_ok tx_carrier_off

7.6 tx_single

Phat dung mot frame roi dung:

cmd_ok tx_single

7.7 tx_status

In snapshot TX hien tai:

tx_status tx_enabled=<0|1> carrier_test=<0|1> tx_frames=<N> tx_frame_id=<N> bit_rate=100000 len=<N> payload=<HEX...> checksum=<XX> frame=<HEX...>

8. Cap nhat frame runtime an toan

Khi doi payload bang UART, firmware khong ghi chong len du lieu TX dang duoc ISR doc truc tiep. Cach lam:

  1. Build frame moi vao buffer tam.
  2. Vao critical section ngan bang __disable_irq().
  3. Copy frame moi vao buffer TX.
  4. Reset chi so bit ve dau frame.
  5. __enable_irq().

Muc tieu la giu ISR ngan va tranh frame bi doc do dang cap nhat.

11. Luong kiem thu

Khi kiem thu TX_ONLY, nen quan sat:

  1. tx_frames tang deu theo thoi gian.
  2. tx_frame giu dung payload mac dinh hoac payload vua set.
  3. cmd_ok tx_payload ... phan hoi dung checksum.
  4. Tren oscilloscope:
    • carrier 1 MHz van co tren PA8
    • OOK bat/tat theo bit 1/0
  5. Khong co log loi tu ISR.

12. Ket luan

  • phan cung carrier va timer giu co dinh,
  • ISR chi lam viec toi thieu,
  • frame duoc build theo protocol thong nhat,
  • UART command xu ly o foreground,
  • runtime update frame dung critical section ngan,
  • logging duoc tach rieng khoi ISR.

Dieu nay giup firmware giu duoc tinh on dinh realtime dong thoi van du linh hoat de GUI dieu khien TX va theo doi trang thai.
